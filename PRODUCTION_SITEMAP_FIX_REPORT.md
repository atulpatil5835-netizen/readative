# Production Sitemap Fix Report

Generated: 2026-06-18

## Root Cause

Production had two independent blockers.

1. `FUNCTION_INVOCATION_FAILED` was caused by Node ESM module resolution in the Vercel functions. Vercel compiled the TypeScript API files to ESM JavaScript, but the generated files kept extensionless relative imports such as `./_seoData` and `../src/utils/seoTaxonomy`. Node ESM does not resolve those to `.js` files, so the function failed at module load with `ERR_MODULE_NOT_FOUND` before any handler code or fallback response could run.

2. `/sitemap.xml` was masked by `public/sitemap.xml`. Vercel routes static filesystem assets before the rewrite to `/api/sitemap.xml`, so production served the old 28-URL static sitemap instead of the Firestore-backed dynamic sitemap.

## Fix Summary

- Added explicit `.js` specifiers to serverless relative imports so the built Vercel ESM functions load in Node.
- Removed the static `public/sitemap.xml` asset so `/sitemap.xml` resolves to `/api/sitemap.xml`.
- Added crawlable SmartTalk entry URLs at `/smarttalks/{id}`.
- Added all SmartTalk entries to the dynamic sitemap.
- Updated `/posts` and `/smarttalks` discovery output to link to the crawlable SmartTalk entry URLs.
- Added a Vercel rewrite for `/smarttalks/:id` to the SmartTalk serverless handler.
