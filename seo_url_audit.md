# Release H5 SEO URL Audit

Status: implementation and validation complete.
Date: 2026-07-10

## Objective

Implement premium, human-readable canonical URLs for public Readative posts and SmartTalk discussions while preserving existing document IDs, legacy routes, product behavior, Firestore schema, and existing crawlability surfaces.

## Canonical URL Contract

- Posts: `/posts/<seo-slug>--<documentId>`
- SmartTalk discussions: `/smarttalk/<seo-slug>--<documentId>`
- Document IDs remain embedded in every item URL for uniqueness.
- Slugs are generated by one shared utility: `src/utils/seoUrls.ts`.
- Legacy post inputs remain supported: `/post/<id>`, `/post/<slug>--<id>`, `/knowledge/<id>`.
- Legacy SmartTalk inputs remain supported: `/smarttalk/<id>`, `/smarttalks/<id>`, `/smarttalks/<slug>--<id>`, and query-style `/smarttalk?id=<id>`.

## Shared Slug Engine

Implemented in `src/utils/seoUrls.ts`.

- Lowercases text.
- Normalizes unicode with `NFKD`.
- Removes combining marks and punctuation.
- Converts separators and whitespace to hyphens.
- Collapses duplicate hyphens.
- Trims leading/trailing hyphens.
- Produces deterministic fallback slugs when a caller only has an ID.
- Extracts the document ID from both `slug--id` and plain legacy ID segments.

## URL Producers Audited

- Knowledge Feed focused post URL generation now uses `buildPublicPath("knowledge", { focusedEntryId, seoTitle })`.
- Knowledge Card title anchors and share/copy URLs now use canonical post URLs.
- Continue Reading, Related Posts, Related SmartTalk, and Knowledge Journey actions now use canonical URLs where titles/content are available.
- Explore post rows, topic collections, active discussions, search results, and Explore structured data now use canonical post and SmartTalk URLs.
- Profile shared-post structured data now emits canonical post URLs.
- Profile saved SmartTalk opens now pass question content for canonical SmartTalk slugs.
- My Notes Continue Reading passes post title when available.
- SmartTalk focused pages, related questions, list rows, schema URLs, FAQ URLs, and discussion URLs now use canonical SmartTalk URLs.
- Sitemap entries now use canonical post and SmartTalk paths.
- Server-rendered `/posts` discovery links now use canonical post and SmartTalk anchors.
- Server-rendered post and SmartTalk documents now render canonical related/continue links.
- OpenGraph, canonical tags, Twitter URL-equivalent metadata, breadcrumbs, and JSON-LD now derive from the same canonical path per focused server document.

## URL Consumers Audited

- Client router parses new post URLs, old post URLs, new SmartTalk URLs, old singular SmartTalk URLs, and existing plural SmartTalk URLs.
- Route normalization preserves old hash routes and converts them into path routes.
- Focused post pages replace fallback/legacy client URLs with title-based canonical paths after the document loads.
- Focused SmartTalk pages replace fallback/legacy client URLs with title-based canonical paths after the question loads.
- Vercel rewrites route canonical post URLs to `api/post`.
- Vercel rewrites route canonical SmartTalk URLs to `api/smarttalks`.
- Legacy `/post/:id` and `/smarttalks/:id` rewrites mark requests as legacy so the server can return 301 to canonical.
- Static `_redirects` mirrors the same canonical and legacy rewrite support.

## SEO Surfaces Audited

- Canonical tags: focused server documents use the new canonical URL.
- OpenGraph URLs: focused server documents use the new canonical URL.
- Twitter card metadata: retained and paired with the same focused document context.
- JSON-LD: Article, DiscussionForumPosting, FAQPage, BreadcrumbList, CollectionPage, and ItemList URLs use the canonical path helpers.
- Sitemap: post and SmartTalk entries use canonical slug URLs.
- Robots: canonical URL families are crawl-eligible.
- Discovery index: emits crawlable canonical anchors for every public post and SmartTalk discussion.

## Safety Notes

- No Firestore schema changes.
- No feed ranking changes.
- No SmartTalk business logic changes.
- No Notebook, Auth, Notifications, Analytics, Cookies, Profile, or desktop layout behavior changes.
- Existing IDs remain the data source of truth.
- Legacy routes remain functional and redirect-compatible.
- `/posts` remains the crawlable post discovery index.
- `/smarttalks` remains the crawlable SmartTalk discovery index.

## Validation Evidence

- Build, TypeScript, strict TypeScript, SEO verifier, and diff whitespace gates passed.
- SEO verifier found 336 canonical post URLs, 109 canonical SmartTalk URLs, 512 sitemap URLs, zero missing post URLs, zero missing SmartTalk URLs, and zero duplicate sitemap URL groups.
- Handler smoke confirmed canonical post and SmartTalk URLs return 200.
- Handler smoke confirmed legacy post, legacy plural SmartTalk, legacy singular SmartTalk, and query-style SmartTalk inputs return 301 to canonical URLs.
- Browser smoke on the production bundle confirmed canonical tags, OpenGraph URLs, refresh, back, and forward for representative post and SmartTalk routes.
- Auth-personalized Bookmarks and Notifications were code-path/static verified; no signed-in browser session with saved/notification data was available for E2E interaction.
